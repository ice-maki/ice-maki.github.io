准备

下载题目
修改launch.sh，添加-s /,用于gdb调试,修改kalsr为nokalsr,修改–foreground增加调试时间

函数分析

init_func

init_func：通过虚拟文件/proc/pwn_device与内核交互，权限438->0666->rw-rw-rw-，用户可读写

sioctl

sioctl:cmd（第二参数）等于32时改变MaxBuffer。

sread

sread函数:只接收两个参数，但在调用 copy_to_user 时，第三个参数（用户传入的读取长度）未经检查直接使用，导致可以读取超出栈缓冲区范围的数据。

利用代码grep "sread.cold" /proc/kallsyms找到sread_cold函数地址并使用gdb断点调试可看到sread函数栈结构如下.

当用户设置buf[19]，同时执行read(fd,buf,sizeof(buf))就正好可以读取到canary->buf[16]和rip->buf[18]. 利用rip计算出内核基址,内核基址`ffffffff81000000 T startup_64`，偏移为0x23e347

swrite

swrite函数:如果a3(写入数据大小)<=maxbuf，则将a2(写入数据)复制到内核栈上。maxbuf由ida可见，值40h
因为canary位于v4[16]，所以在构造溢出payload时，从索引16开始覆盖

在swrite_cold断点可看到swrite栈帧如下
通过sioctl将maxbuf修改为>=0x90，可以栈溢出覆盖rip，不过因为要构造rop链，所以这里设置maxbuf为0x200

接下来就是构造rop获取root权限的shell

题目防护

smep（内核不能跳转到用户代码）
smap（内核不能直接读写用户内存）
kaslr(内核地址随机化)->利用read漏洞绕过
kpti（页表分离）->利用kpti trampoline绕过，位于swapgs_restore_regs_and_return_to_usermode()函数+0x16

构造payload

设置函数

第一步:设置两个函数，一个提前保存cs，rflags，rsp，ss值,一个获取shell

因为rop会覆盖内核函数返回地址导致无法正常通过sysret返回用户态，所以需要利用iretq返回用户态，
iretq的作用是弹出rip，cs，rflags，rsp，ss，所以需要提前保存cs,rflag,rsp,ss的值。
rip设置为指向一个获取shell的函数，这样从内核回到用户态就会直接打开一个root权限的shell。

计算所需内核函数地址

第二步，利用read获取的内核基地址，计算所需函数地址(偏移=根据kallsyms查到的地址-基地址)
prepare_kernel_cred(0) 用于创建一个 root 权限的凭证结构，返回值在 rax 中，然后通过commit_creds()将其应用于当前进程。
kpti_trampoline ->绕过kpti

找gadget

第三步，构造rop，找gadget，先看看有没有mov rdi，rax；ret（因为kernel_prepare_cred(0)的返回值是放在rax里的），
因为rax不为零，不能选有test和jne的gadget了，所以这里我们选0xffffffff813b3504 : mov rdi, rax ; jne 0xffffffff813b34f1 ; xor eax, eax ; ret这个gadget。

这里的话需要一个gadget设置ZF标志位为1，因为我们不想执行jne。选个用不到的寄存器rdx，看看有没有test rdx, rdx，正好找到一个0xffffffff810a54ab : test rdx, rdx ; jne 0xffffffff810a5491 ; ret。

这里只需要再找个pop rdx设置一下rdx的值为0就可以让ZF=1，jne不执行，找到一个0xffffffff81034b72 : pop rdx ; ret

漏了一个pop rdi;ret
